# QueueSmart Login / Registration / User Dashboard

## Project Overview

This project is my part of Assignment 3. I completed the following sections:

* Login
* Registration
* User Dashboard

The project uses:

* Node.js
* Express.js
* HTML
* CSS
* JavaScript
* In-memory data storage

This means user data is stored while the server is running. If the server is stopped and started again, the users created before will be reset.

---

## What This Part Includes

### Login

Users can:

* enter email and password
* log in with a registered account
* see an error message if the login is invalid

### Registration

Users can:

* create a new account
* enter first name, last name, email, and password
* receive validation for missing or incorrect fields

### User Dashboard

After logging in, users can view:

* current queue position
* people ahead
* estimated wait time
* service type
* notifications
* account history

## How to Run the Project

### 1. Open the project folder

Open the project in VS Code.

### 2. Open the terminal

In VS Code, open the terminal.

### 3. Install dependencies

Run:

```bash
npm install
```

### 4. Start the server

Run:

```bash
npm start
```

You should see something similar to:

```bash
Server running on http://localhost:3000
```

### 5. Open the website

Open your browser and go to:

```text
http://localhost:3000
```

This should open the login page.

## Important Note About Data Storage

This project uses **in-memory storage**.

That means:

* user accounts are stored only while the server is running
* if the server is restarted, saved users are reset

This is acceptable for this assignment because a database is not required for this part.

## Troubleshooting

### Problem: Page does not open

Make sure the server is running with:

```bash
npm start
```

### Problem: Cannot register or log in

Check:

* that the server is still running
* that the email and password were typed correctly
* that the browser console does not show an error

### Problem: Dashboard does not load

Check:

* whether login was successful
* whether `userId` is stored in localStorage
* whether the backend routes are running correctly

### Problem: Tests do not run

Make sure dependencies are installed:

```bash
npm install
```

Then run:

```bash
npm test
```
